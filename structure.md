# Claude Code для команды разработки

## Глава 1: Проблема

- Разработчики тратят огромное количество времени на рутинные задачи
- Чтение и понимание чужого кода — одна из самых затратных активностей
- Переключение контекста между задачами снижает продуктивность
- Документация устаревает, onboarding новых разработчиков занимает недели
- Кодовые базы растут: наш проект homepage_next — это 4 900+ Python-файлов, 2 подпроекта (ostecPlatform и smartLine), генератор OpenAPI-спецификаций

## Глава 2: Что такое Claude Code

- CLI-инструмент от Anthropic, работающий прямо в терминале / IDE
- Агент, который понимает контекст всего проекта
- Умеет читать файлы, искать по кодовой базе, запускать команды, редактировать код
- Управляется через CLAUDE.md — файл с инструкциями для ИИ, описывающий архитектуру проекта, правила и команды
- Поддерживает "навыки" (skills) — специализированные сценарии (генерация фронтенда, код-ревью, коммиты и др.)

## Глава 3: Реальный пример — проект homepage_next

### Что за проект
- Система генерации OpenAPI 3.1.0 спецификаций из исходного кода
- Автоматически парсит RST-документацию, Python docstrings, routes config, HTTP-семплы
- 2 подпроекта: ostecPlatform (189 файлов) и smartLine (4 700+ файлов)
- Генератор: open_api_generator.py — 2 394 строки кода
- Спецификация генератора: openApiGen2.md — 2 312 строк

### Как CLAUDE.md помогает
- Файл CLAUDE.md описывает критические правила: «НИКОГДА не редактировать src/»
- Содержит команды запуска для каждого подпроекта
- Описывает архитектуру: где генератор, где исходники, где выходные файлы
- Новый разработчик (или ИИ-агент) сразу понимает структуру проекта

### Сложность проекта
- Python-бэкенды: Flask/Gevent, PostgreSQL, LDAP, JWT, Kerberos
- smartLine: 11+ специализированных серверов (Fuji, Hanwha, Samsung, AOI, печи, принтеры, маркеры)
- Интеграция с ERP-системой (URM), паспортная система
- Документация на русском языке через Sphinx
- Docker-контейнеризация

## Глава 4: Возможности Claude Code в работе

### Навигация и понимание кода
- Мгновенный поиск по всей кодовой базе (Glob, Grep)
- Чтение и объяснение любого файла
- Понимание зависимостей между модулями
- Ответы на вопросы: «Как работает аутентификация?», «Где обрабатываются HTTP-запросы?»

### Написание и редактирование кода
- Генерация нового кода с учётом существующих паттернов проекта
- Рефакторинг с сохранением стиля кодовой базы
- Автоматическое создание тестов
- Исправление багов с пониманием контекста

### Автоматизация рутины
- Генерация коммитов с осмысленными сообщениями
- Создание pull request-ов с описанием изменений
- Код-ревью
- Генерация документации

### Работа с несколькими файлами одновременно
- Параллельный запуск подагентов для независимых задач
- Изменение нескольких файлов в рамках одной задачи
- Координация изменений между связанными модулями

## Глава 5: Безопасность и контроль

- Разработчик видит и подтверждает каждое действие агента
- Режимы разрешений: полный контроль или автоматическое выполнение
- CLAUDE.md задаёт границы: что можно редактировать, а что нельзя
- Пример: правило «NEVER edit anything inside src/» защищает исходный код проектов
- Все изменения — через git, можно откатить в любой момент
- Код не отправляется на внешние серверы для обучения модели

## Глава 6: Интеграция в рабочий процесс

- Работает в терминале и в VS Code
- CLAUDE.md коммитится в репозиторий — знания о проекте доступны всей команде
- Каждый разработчик получает ИИ-ассистента, который знает контекст проекта
- Onboarding новых сотрудников: ИИ объясняет архитектуру, находит нужные файлы
- Ускорение код-ревью: автоматический анализ изменений

## Глава 7: Результаты и метрики

- Снижение времени на понимание незнакомого кода: часы → минуты
- Ускорение рутинных задач (коммиты, PR, документация) в 5–10 раз
- Быстрый onboarding: новый разработчик продуктивен с первого дня
- Единый источник знаний о проекте через CLAUDE.md
- Меньше ошибок за счёт автоматического следования правилам проекта
